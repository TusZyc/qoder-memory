## Beta KB API 集成

### 背景
旧 KB (kb.ceve-market.org) 从服务器端访问会返回 302 重定向，导致无法在后端提取 ESI hash。发现 beta.ceve-market.org 有可用的 REST API，服务器端访问正常返回 200。

### 实现
1. **ESI Hash 提取升级** (`extractEsiHash()`)：
   - 主路径：Beta KB `/app/kill/{id}/info` → protobuf 响应 → 正则提取 40 字符 hex hash
   - 备用：旧 KB HTML 解析
   - 缓存 86400s TTL

2. **击杀列表 API** (`fetchBetaKillList()`)：
   - Beta KB `/app/list/pilot/{id}/kill` → protobuf 响应 → 50 条/页
   - 返回完整元数据：kill_id, victim_name, ship_name, system_name, kill_time, total_value, esi_hash
   - 自动缓存每条记录的 ESI hash

## Protobuf 解码器

Beta KB API 返回 `application/alicegearaegis` (二进制 protobuf)，实现了完整的 PHP protobuf 解码工具：

- `pbDecodeVarint(string $data, int $offset, int &$newOffset): int` — 变长整数解码
- `pbParseMessage(string $data): array` — 解析消息为 [field, wire, value] 数组
- `pbGetVarint(array $fields, int $fieldNum): ?int` — 提取 varint 值
- `pbGetString(array $fields, int $fieldNum): ?string` — 提取字符串（处理嵌套 UTF-8）
- `pbGetDouble(array $fields, int $fieldNum): ?float` — 提取 double 值

### 字段映射
| 字段号 | 类型 | 内容 |
|--------|------|------|
| F1 | varint | kill_id |
| F2 | message | victim (嵌套含名称) |
| F8 | message | ship (嵌套含名称) |
| F9 | message | kill_time (嵌套时间戳) |
| F10 | message | solar_system (嵌套含名称) |
| F13 | double | total_value (ISK) |
| F16 | message | esi_hash (嵌套含 40 字符 hex) |
