## 2026-03-18 Token 过期问题排查与 OAuth 流程详解（第四部分）

### 问题现象

- 网站提示「授权已过期」
- 首页服务器状态无法显示
- 访问授权页面被跳转到仪表盘

### 问题排查

1. **日志分析**：ESI 返回 `invalid_grant` 错误
   ```
   "error":"invalid_grant"
   "error_description":"Invalid refresh token. Character grant missing/expired."
   ```

2. **数据库检查**：所有用户的 refresh_token 都只有 24 个字符
   - 初步怀疑是存储截断问题
   - 实际是 EVE 国服的正常行为

3. **根本原因**：
   - Laravel Session 登录状态与 EVE OAuth Token 是独立的
   - AuthController::guide() 检查已登录就跳转 dashboard
   - refresh_token 失效后用户被困住

### 修复内容

1. **AutoRefreshEveToken 中间件改进**
   - 当 refresh_token 失效（400/401 错误）时强制登出用户
   - 清除数据库中的 token 信息
   - 显示提示「EVE 授权已过期，请重新授权」

2. **AuthController 添加调试日志**
   - 记录 OAuth 响应中 token 的实际长度

### EVE OAuth 2.0 流程详解

```
用户点击授权链接
    ↓
EVE 登录页面授权
    ↓
重定向返回 authorization code（一次性，5-10分钟有效）
    ↓
用户复制 URL 粘贴到网站
    ↓
后端用 code 换取 token
    ↓
返回 access_token（~20分钟）+ refresh_token（7-30天）
```

### EVE 国服 OAuth 特性验证

**测试结果**：刷新 token 时 refresh_token 保持不变

| 对比项 | 刷新前 | 刷新后 |
|--------|--------|--------|
| access_token | 旧的 | ✅ 新的 |
| refresh_token | `ibNf/9bMq0uYvD6VLQR2zA==` | 相同 |
| expires_in | - | 1199 秒 |

**结论**：
- EVE 国服不使用 Token Rotation 机制
- refresh_token 在有效期内保持不变（只有 24 个字符是正常的）
- 之前的问题是 refresh_token 本身过期失效

### 新增经验

- EVE 国服 refresh_token 只有 24 个字符（非存储问题）
- Laravel Session 和 OAuth Token 状态需要同步处理
- Token 失效时应强制登出，避免用户被困在死循环
