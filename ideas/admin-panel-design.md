# EVE ESI 后台管理页面设计方案

**创建日期：2026-03-18**

---

## 一、背景

EVE ESI 网站目前功能已较为完善（SSO 登录、仪表盘、技能、资产、KM、市场、角色信息、旗舰导航等），但缺少后台管理能力。本次设计旨在为网站增加「军团管理」和「站点管理」两个维度的后台功能。

---

## 二、功能规划

### 2.1 军团管理后台

面向军团管理者 / FC 使用，基于角色的 ESI 授权数据。

#### 2.1.1 军团概况 / 仪表盘

- 军团基础信息
  - 军团名称、Ticker、Logo、简介
  - 所在势力、成员数量、创建时间
- 在线 & 活跃
  - 近期在线人数统计（按天/周）
  - 活跃时区分布（简单按小时段统计）
- 成员结构
  - 职位分布（CEO、Director、普通成员等）
  - 舰船类别偏好（战列/巡洋/后勤等）
- 快捷信息
  - 待处理的入团申请数
  - 近期重要 Killmail 列表

#### 2.1.2 成员管理（核心）

- 成员列表
  - 成员名、角色ID、职位/军衔
  - 最近登录时间、入团天数
- 成员详情
  - 复用现有「角色信息页」
  - 资产概览、技能概览、击杀/死亡统计
- 成员状态管理
  - 标签系统：重点关注、准备踢出等
  - 手工备注（常玩时区、擅长舰船）

#### 2.1.3 权限 / 角色分级（军团内部）

- 角色类型
  - 军团管理员：全数据访问 + 站点配置权限
  - 军团指挥（FC）：舰队、战斗数据查看
  - 普通成员：自己的数据 + 部分军团统计
- 权限控制点
  - 页面级权限
  - API 级权限

#### 2.1.4 军团资产 / 资产统计

- 资产汇总
  - 全军团资产估值
  - 分类统计：舰船、模块、矿物、LP、ISK
- 舰队资产结构
  - 常用舰船数统计
  - 关键旗舰数量（泰坦、超航、JDC 相关角色）

#### 2.1.5 军团活动 / 战报

- 军团战报列表
  - 最近战斗：胜利/失败关键战
  - 筛选：时间区间、舰队标签、指挥官
- 战斗统计
  - 总击杀数、总损失数、ISK 统计
- 军团成员参与度
  - 按成员统计参与战斗次数、出勤率

#### 2.1.6 旗舰导航配置

- 常用目的地/航线管理
- 舰队预设（集结点 + 舰种要求 + 补给点）
- 权限控制

---

### 2.2 站点管理后台

面向站点管理员（图斯）使用，偏运维 + 配置方向。

#### 2.2.1 用户 & 角色管理（站点级）

- 站点用户列表
  - 所有登录过站点的角色
  - 关联军团、最后登录时间、使用功能情况
- 站点角色
  - Site Admin：全权限
  - Corp Admin：对应某个军团的管理员
  - 普通用户：默认
- 角色绑定 / 切换
  - 站点用户与军团管理员身份绑定
  - 军团撤权时一键取消绑定

#### 2.2.2 SSO / 授权管理

- 应用授权状态总览
  - ESI Client ID、回调地址展示
- Token 状态监控
  - 活跃 refresh token 数量、过期情况
  - 400 invalid_grant 错误次数统计
- 一键处理
  - 强制注销指定角色
  - 标记异常账号

#### 2.2.3 日志 & 运行监控

- 应用日志浏览
  - 按天查看 Laravel 日志
  - 搜索：角色ID / 请求路径
- API 调用统计
  - ESI 调用次数、错误率
  - 站点内部 API 耗时分布
- 队列 / 缓存状态
  - 队列任务数量
  - 缓存命中率

#### 2.2.4 配置管理

- 站点基本配置
  - 站点标题、副标题
  - 是否开放游客仪表盘
  - 是否开放注册
- 功能开关（Feature Flags）
  - 资产估值：alpha / beta / 关闭
  - 军团管理：启用 / 关闭
  - 实验功能开关
- 缓存控制
  - 一键清理特定缓存

#### 2.2.5 内容 / 文案管理（可选）

- 公告管理
  - 首页/仪表盘顶部紧急公告
- 文案配置
  - 重要提示文案可修改

## 四、登录与鉴权方案

### 4.1 核心原则

**不需要新建账号系统**，在现有 EVE SSO + Cookie 登录基础上叠加权限层即可。

现有认证体系：
- 身份来源：EVE SSO（OAuth）
- 会话载体：Laravel Session + Cookie
- 站点账号：users 表记录（已有）

### 4.2 登录流程

1. 用户访问 `/admin` 或 `/corp-admin`：
   - 未登录 → 跳转 EVE 授权入口
   - 已登录 → 进入第 2 步

2. 已登录用户访问后台：
   - 检查 ESI token 健康状态
     - 若刷新失败或 `invalid_grant` → 强制登出 + 提示重新授权
   - 检查站点角色权限
     - `user.role === 'site_admin'` → 站点后台
     - `user.role === 'corp_admin'` → 军团后台

3. 权限不足：
   - 返回 403 或重定向到普通仪表盘

### 4.3 实现方案

#### 方案一：极简版（几乎不动数据库）

适合先试水、只给管理员使用。

```php
// config/admin.php
return [
    'site_admins' => [
        123456789, // 管理员角色ID
    ],
    'corp_admins' => [
        987654321, // 军团长ID
    ],
];
```

中间件：
- `EnsureSiteAdmin`：检查 character_id 是否在 site_admins
- `EnsureCorpAdmin`：检查 corp_admins 或 corp_id

路由：
```php
Route::middleware(['auth', 'site_admin'])->group(function () {
    Route::get('/admin', ...);
});

Route::middleware(['auth', 'corp_admin'])->group(function () {
    Route::get('/corp-admin', ...);
});
```

#### 方案二：完整版（基于 users 表角色系统）

适合多用户场景。

1. users 表加字段：
   - `role`: `'user' | 'corp_admin' | 'site_admin'`
   - 可选：`corp_id`

2. 后台提供角色管理功能（仅 site_admin 可见）

3. 中间件读 `user.role` 判断权限

### 4.4 安全要点

1. **单一登录入口**：全站只用 EVE SSO 一种登录方式
2. **路由级严格鉴权**：后台路由必须挂 `auth` + 角色中间件
3. **Token 健康检查**：进入后台时检查 Refresh Token 状态
4. **操作审计**：敏感操作记录日志（谁、何时、做了什么）

## 五、优先开发顺序

1. **站点管理最小版**
   - 站点用户列表 + 角色（Site Admin / 普通用户）
   - 简单日志查看

2. **军团概况 + 成员列表最小版**
   - 只读：军团信息 + 成员列表
   - 成员详情跳转到现有角色页

3. **逐步扩展**
   - 权限分级（军团管理员 / FC）
   - 资产 / 战报统计
   - 导航配置、功能开关

## 六、技术实现要点

### 6.1 路由规划

```
/admin                    → 站点管理首页
/admin/users              → 用户管理
/admin/roles              → 角色管理
/admin/logs               → 日志查看
/admin/config             → 配置管理

/corp-admin               → 军团管理首页
/corp-admin/members       → 成员管理
/corp-admin/assets        → 军团资产
/corp-admin/battles       → 战报统计
/corp-admin/navigation    → 导航配置
```

### 6.2 中间件

- `auth`：Laravel 内置认证
- `site_admin`：站点管理员权限
- `corp_admin`：军团管理员权限
- `token.healthy`：ESI Token 健康检查

### 6.3 Controller 规划

```
Admin/
├── DashboardController.php      → 站点管理首页
├── UserController.php           → 用户管理
├── RoleController.php           → 角色管理
├── LogController.php            → 日志查看
├── ConfigController.php         → 配置管理
└── TokenController.php          → Token 管理

CorpAdmin/
├── DashboardController.php      → 军团管理首页
├── MemberController.php         → 成员管理
├── AssetController.php          → 军团资产
├── BattleController.php         → 战报统计
└── NavigationController.php     → 导航配置
```
