# 2026-04-10 Codex 自建 SDE 记录

## 本次目标
- 从“装配模拟器”开始验证一条新的数据路线：官方 SDE -> 自建 SDE -> 功能页面专用数据。
- 目标不是把官方 SDE 全量搬进网站，而是提取装配模拟器真正需要的数据，做成更小、更稳定、更容易维护的数据底座。

## 已完成内容
- 新增构建命令：
  - `php artisan fitting:build-official-subset`
- 新增输出数据库：
  - `database/fitting_official.sqlite`
- 服务器部署位置：
  - `/opt/eve-esi/database/fitting_official.sqlite`
- 构建过程不会修改旧的生产装配库 `fitting.sqlite`，方便新旧路线并行验证。

## 自建 SDE 当前包含
- 舰船、装备、弹药、技能、无人机、植入体、子系统等装配相关物品。
- 物品分类、物品分组、市场分类。
- Dogma 属性和效果，也就是装配规则、资源消耗、限制条件等核心数据。
- 本地 `data/eve_items.json` 继续用于补充中文名和更接近游戏内习惯的分类提示。

## 已处理的脏数据
- 用户确认过的一批奇怪武器类物品，很多属于 NPC、旧物品、废案或普通玩家不可用物品，不应出现在模拟装配里。
- 构建规则已收紧：
  - 不给这些物品普通槽位。
  - 不让它们进入普通装配列表。
- 舰船模式切换类物品也被排除，因为它们不是正常可安装装备。

## 页面接入进度
- 装备分类树开始读取自建 SDE。
- 装备搜索接口也切到自建 SDE 优先。
- 装备列表和搜索结果统一返回槽位、CPU、能量栅格、校准值、稀有度、图片地址等信息。
- 点击装备可自动安装到合适槽位。
- 新增无人机舱。
- 新增“适合当前舰船”和“资源足够”两个筛选。
- 新增子系统、改装件尺寸、旗舰/超大型装备等基础规则。

## 重要结论
- “官方 SDE -> 自建 SDE -> 装配模拟器数据子集”这条路线已经验证可行。
- 后续重点不再是“能不能做自建 SDE”，而是继续把装配规则解释层做稳，避免旧数据和新数据混用导致结果不一致。
