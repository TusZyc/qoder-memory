## 本地数据集成（ceve-market.org）
### 方案
- 使用 ceve-market.org 的 evedata.xlsx 替代大量 ESI API 调用
- xlsx 包含 6 张表：物品(28K)、星域(109)、星座(1.1K)、星系(8.2K)、NPC站(5.2K)、玩家建筑(247)

### 实现
1. **Python 脚本** `scripts/update_evedata.py`：
   - 下载 xlsx，用 openpyxl 解析，生成 eve_names.json (43,305条, 1.7MB) + eve_station_systems.json (5,464条, 110KB)
   - 已部署到服务器并成功运行

2. **EveDataService 改造**：
   - 数据源从 items.json（旧 [{id,name}] 格式）改为 eve_names.json（{id: name} 格式）
   - 新增 getStationSystemMap() 方法
   - updateData() 从不完整的 xlsx 解析改为调用 Python 脚本
   - 缓存 key 从 eve_item_database 改为 eve_names_database，TTL 增至 2 小时

3. **AssetDataController 改造**：
   - getLocationInfo()：NPC站/玩家建筑先查本地数据，有名称+星系ID直接用，避免 ESI 调用
   - getSolarSystemNames()：星系名先查本地 eve_names.json，避免 ESI 调用

4. **自动更新**：
   - UpdateEveData artisan 命令调用 EveDataService::updateData()
   - host crontab 每周一 02:00 执行
