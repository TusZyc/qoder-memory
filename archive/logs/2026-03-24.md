### 2. 克隆体空间站仍显示英文

**原因**：PHP静态变量缓存了空数据，即使文件恢复也不生效

**修复**：重启PHP-FPM容器清除内存缓存
```bash
docker restart eve-esi-app
docker exec eve-esi-app php artisan cache:clear
```

### 3. 市场搜索功能不工作

**现象**：
- 搜索框搜不出东西
- 分类框不显示搜索结果
- 搜索期间手动选择分类也无法加载物品订单信息

**原因分析**：
1. 清空Redis缓存后，`eve_names_database` 缓存丢失
2. `getMarketGroupsTree()` 方法没有利用静态文件，每次缓存miss都尝试从ESI重建
3. ESI API调用耗时超过30秒导致超时（500错误）
4. 搜索API依赖 `buildTypeCategoryMap()` → `getMarketGroupsTree()`，连锁超时

**修复**：修改 `MarketService.php` 的 `getMarketGroupsTree()` 方法，优先从静态文件加载：

```php
public function getMarketGroupsTree(): array
{
    return Cache::remember('market_groups_tree', 86400 * 7, function () {
        // 优先从静态文件加载（速度快且可靠）
        $staticFile = public_path('data/market_groups.json');
        if (file_exists($staticFile)) {
            $content = file_get_contents($staticFile);
            $data = json_decode($content, true);
            if (is_array($data) && !empty($data)) {
                Log::info('Market groups tree loaded from static file');
                return $data;
            }
        }
        
        // 静态文件不存在或为空，从ESI获取
        Log::info('Market groups tree cache miss, rebuilding from ESI...');
        return $this->buildMarketGroupsTreeFromApi();
    });
}
```

## 经验教训

1. **代码同步时需谨慎检查删除操作**：大文件（如数据JSON）可能被误删，提交前应仔细检查 `git status`
2. **PHP静态变量缓存**：需要重启PHP-FPM才能清除内存中的静态变量缓存
3. **Redis缓存与静态文件配合**：关键数据应优先从静态文件加载，避免依赖外部API导致超时
